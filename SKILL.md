# 世界杯比赛比分预测

## 核心功能

自动获取当天比赛数据，结合9个维度的信息源进行AI综合分析，输出比分预测。

## 触发条件

当用户提到以下关键词时触发：
- 今天的比赛
- 明天的比赛
- 下一轮的比赛
- 预测比分
- 世界杯赛程
- 足球比赛预测
- 比赛分析

## 时区说明

**用户位于北京时间（UTC+8），所有时间判断和输出必须基于北京时间。**

### 时间判断规则

**所有时间均指代用户当前时区时间（北京时间，东八区 UTC+8）。**

1. **指定日期**：如果用户明确提供了比赛时间（如"6月20日的比赛"、"明天的比赛"、"今天的比赛"），则严格按照用户提供的北京时间日期进行查询。
2. **未指定日期**：如果用户**没有**提到具体时间（如仅说"预测比赛"、"世界杯预测"），**必须先询问用户**具体想预测哪天的比赛，并主动询问是否需要预测**明天**的比赛（例如："请问您想预测哪一天的比赛？需要帮您预测明天的比赛吗？"）。

| 用户表述 | 北京时间含义 | 查询日期 |
|----------|--------------|----------|
| 指定具体日期(如"6月20日") | 该日期的北京时间00:00-23:59 | 用户指定日期 |
| "今天的比赛" | 今天北京时间00:00-23:59 | 今天 |
| "明天的比赛" | 明天北京时间00:00-23:59 | 明天 |
| "下一轮的比赛" | 最近一场未开始的比赛 | 动态查询 |
| 未指定时间 | **必须先询问用户** | 询问后决定 |

### 时区转换

ESPN显示的是美东时间（ET），需要转换为北京时间：

```
北京时间 = 美东时间 + 12小时
```

**注意：转换后如果跨天，需要使用第二天的日期。**

示例：
- 3:00 PM ET = 次日 3:00 AM 北京时间
- 6:00 PM ET = 次日 6:00 AM 北京时间
- 9:00 PM ET = 次日 9:00 AM 北京时间
- 12:00 AM ET = 当天 12:00 PM 北京时间

### ESPN日期查询

ESPN使用美东时间的日期，北京时间比美东时间早12小时。

**正确查询方法：**

1. 先确定北京时间的目标日期范围（如明天 = 6/17 00:00 至 6/18 00:00）
2. 转换为美东时间范围（北京时间 - 12小时）：
   - 北京时间 6/17 00:00 = 美东时间 6/16 12:00
   - 北京时间 6/18 00:00 = 美东时间 6/17 12:00
3. 用美东时间的日期去 ESPN 查询

**结论：** 查询北京时间的"明天"，需要查询 ESPN 的**两个日期**（美东日期 -1 和 美东日期）。

示例：用户问"明天（6月17日）的比赛"
- 北京时间范围：6/17 00:00 - 6/18 00:00
- 美东时间范围：6/16 12:00 - 6/17 12:00
- 查询 ESPN 6/16 + ESPN 6/17

## 工作流

```
1. 检查用户输入是否包含具体日期。如果没有，必须先询问用户是否需要预测明天的比赛，并等待回复。
2. 根据用户表述确定查询日期（基于北京时间，UTC+8）
3. 获取该日期的比赛列表
4. 过滤比赛：只保留目标日期的比赛，排除 "+1D"（后天）的比赛
5. 将美东时间转换为北京时间
6. 对每场比赛并行获取6类数据
7. AI综合分析，按权重计算预测
8. 输出预测结果（简洁/详细模式）
```

### 比赛过滤规则

从 ESPN 获取比赛列表后，需要根据日期标签进行过滤：
- **保留**：标记为 "Today"（今天）、"Tomorrow"（明天）或目标日期的比赛
- **排除**：标记为 "+1D"（后天）或更远日期的比赛
- **注意**：ESPN 页面可能同时显示多天的比赛，必须严格按日期过滤，只处理目标日期的比赛

## 数据获取

**重要：综合使用 chrome-devtools-mcp 和 WebSearch 工具获取数据。**
由于部分体育网站存在强反爬措施，请严格按照以下工具分配获取数据，以确保稳定性。

chrome-devtools-mcp 提供以下工具：
- `navigate_page` - 导航到URL
- `take_snapshot` - 获取页面DOM快照（推荐，结构化数据）
- `take_screenshot` - 截图（用于视觉验证）
- `evaluate_script` - 执行JavaScript提取数据
- `new_page` - 打开新标签页
- `wait_for` - 等待特定文本出现

**关于 SPA (单页应用) 网站的数据获取：**
`chrome-devtools-mcp` 是通过真实的 Chrome 浏览器来渲染页面的，因此**完全支持 SPA 网站**。它会等待 JavaScript 执行并渲染出最终的 DOM 结构。
- 关键在于使用 `wait_for: ["特定文本"]`。在导航到 URL 后，必须等待页面上出现只有数据加载完成后才会显示的特定文本（例如 "MATCH", "Standing", "Result" 等）。
- 只有当 `wait_for` 成功后，再调用 `take_snapshot` 或 `evaluate_script`，这样就能确保拿到的是渲染后的完整数据，而不是空壳。

### Step 1: 获取比赛列表

使用 chrome-devtools-mcp 获取ESPN比赛列表：

```
# 1. 打开新页面
new_page: https://www.espn.com/soccer/schedule/_/date/{YYYYMMDD}/league/fifa.world

# 2. 等待页面加载
wait_for: ["MATCH"]

# 3. 获取页面快照提取比赛数据
take_snapshot
```

**过滤规则**：
- 保留标记为 "Today"、"Tomorrow" 或目标日期的比赛
- 排除标记为 "+1D" 的比赛

### Step 2: 获取比赛数据

对每场比赛，按以下方式获取6类数据：

| 数据 | 推荐工具 | 数据源 | 用途 |
|------|----------|--------|------|
| 赔率 | chrome-devtools-mcp 或 WebSearch | ESPN列表 或 搜索 "{主} vs {客} odds" | 博彩赔率(25%) |
| FIFA排名 | chrome-devtools-mcp | 维基百科 FIFA Ranking 页面 | 排名(15%) |
| 近期战绩 | WebSearch | 搜索 "{队伍} recent match results" | 状态(15%) |
| 历史交锋 | WebSearch | 搜索 "{主} vs {客} head to head history" | 交锋(10%) |
| 阵容信息 | WebSearch | 搜索 "{队伍} injuries suspensions" | 阵容(10%) |
| 预选赛/积分榜 | chrome-devtools-mcp | ESPN积分榜 或 维基百科 | 表现(10%) |

详细URL和JavaScript提取脚本见 `references/data-sources.md`

## 权重模型

| 因素 | 权重 | 说明 |
|------|------|------|
| 博彩赔率 | 25% | 市场共识，信息密度最高 |
| 阵容质量 | 15% | 核心球员伤病/停赛对国家队影响巨大 |
| FIFA排名 | 15% | 长期综合实力基准 |
| 近期战绩 | 15% | 当前状态和士气 |
| 预选赛表现 | 10% | 正式赛事竞技水平 |
| 主客场因素 | 8% | 2026北美地利及气候适应性 |
| 历史交锋 | 5% | 心理优势（国家队交锋间隔久，参考价值有限） |
| 赛程/体能 | 4% | 疲劳程度 |
| 战术克制 | 3% | 风格相克 |

详细评分标准见 `references/weight-model.md`

**注意：在详细模式的分析摘要中，必须在各项指标后使用括号注明该数据的来源（如 `(ESPN)`, `(维基百科)`, `(WebSearch)`）。如果某项数据未能成功获取，必须明确标注 `(未获取到数据)`，并说明该项在计算时采用了默认评分或权重转移。**

| 总分差 | 预测结果 |
|--------|----------|
| > 20 | 大胜 (3-0, 3-1) |
